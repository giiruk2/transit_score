## 4. Redis 캐싱 테스트

- Redis 프로세스: **미연결** (Docker 컨테이너 미실행)
- 1차 호출: 6,319ms / 2차 호출: 2,895ms
- 2차 호출이 빠른 건 OS 네트워크 캐싱이지, Redis 캐시 아님
- `redisCache.ts`의 graceful degradation 정상 작동 (Redis 없어도 서비스 정상)

**결과**: ⚠️ Redis 미연결 — `docker-compose up redis -d` 필요

## 5. Vworld DEM API 테스트 (버그 발견)

### 테스트한 엔드포인트
```
❌ https://api.vworld.kr/req/dem  → 404 Not Found
❌ https://api.vworld.kr/ned/api/dem/getElevation  → 404 Not Found
⚠️ https://api.vworld.kr/req/data  → INCORRECT_KEY (인증키 정보가 올바르지 않습니다.)
```

### 영향
- `routeService.ts`의 `fetchElevation()` 함수가 null 반환
- `calculateToblerWalkTime()` → 고도 데이터 없이 ODsay `sectionTime` 폴백으로 동작
- 토블러 함수(`W = 6 * e^(-3.5*|S+0.05|)`) 실질적으로 미적용

### 원인 분석
1. Vworld `/req/dem` 엔드포인트 자체가 존재하지 않음 (404)
2. Vworld API 키가 해당 서비스에 미등록 (`INCORRECT_KEY`)
3. 정확한 Vworld 고도 조회 엔드포인트 재확인 필요

**결과**: ❌ 버그 — 토블러 함수 실제 미작동, 거리 기반 폴백으로 동작 중
