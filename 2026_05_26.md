## 2. 키보드 포커스 상태

### 배경
거의 모든 인터랙티브 요소에 `:focus-visible` 스타일이 없거나, 일부 input은 `outline-none`으로 포커스 표시 자체를 제거. WCAG AA 키보드 접근성 미달.

### 작업
- [globals.css](frontend/src/app/globals.css)
  - 전역 `:focus { outline: none }` (마우스 클릭 시 outline 안 보임)
  - `button:focus-visible, a:focus-visible, input:focus-visible, ...` 에 accent 색 2px outline + 2px offset
  - `.search-input:focus-visible` 는 별도로 더 굵은 링 + 그림자
- [SearchBar.tsx](frontend/src/components/SearchBar.tsx) — `onFocus`/`onBlur` 의 JS boxShadow 토글 제거하고 클래스 위임
- `outline-none` Tailwind 클래스 제거 위치: `OriginPanel.tsx`, `AuthModal.tsx` ×3, `page.tsx` ×2 (인라인 편집기)

### 가시 변경
- Tab 키 탐색 시 액센트 색 포커스 링이 따라옴
- 마우스 클릭으로 포커스된 경우엔 표시 안 됨 (모던 표준)

## 4. JS hover → CSS hover

### 배경
`onMouseEnter`/`onMouseLeave` 로 `style.opacity`나 `style.background`를 토글하는 패턴이 7쌍 존재. 다른 인라인 style과 충돌 가능성, 불필요한 JS 코드.

### 작업
모두 Tailwind `hover:` 변형으로 교체:

| 위치 | 변경 |
|---|---|
| `page.tsx` (저장 경로 카드, 즐겨찾기 카드) | `hover:opacity-80` |
| `AttractionList.tsx` (그리드/리스트 카드) | `hover:opacity-[0.85]` |
| `SearchBar.tsx` (드롭다운 행) | `hover:bg-[var(--panel-surface)]` |
| `OriginPanel.tsx` (저장 출발지 행) | `hover:bg-[var(--panel-surface)]` |
| `MapViewer.tsx` (컨텍스트 메뉴 버튼) | `hover:bg-[rgba(73,180,222,0.1)]` |

### 가시 변경
- 시각적으론 동일 (`transition-all` 그대로 유지)
- 14줄 JS 제거, 충돌 가능성 해소

## + 교통수단 아이콘 구별성 보완 (사용자 피드백 반영)

### 배경
경로 옵션 리스트에서 `IconBus`/`IconSubway`가 12px 크기에서는 둘 다 "직사각형 + 가로선"으로 보여 구별이 어려움. `IconWalk`도 stride가 잘 안 드러남.

### 작업
[icons.tsx](frontend/src/components/icons.tsx):
- `IconBus` — 바퀴를 **채운 원**(`fill="currentColor"`)으로 변경, 창문 분할선 추가. 가로 직사각형 = 버스 형태
- `IconSubway` — 세로형 **알약 모양**(rx=4)으로 변경 + 하단 트랙선. 버스와 가로/세로 비율로 한눈에 구별
- `IconWalk` — stride(다리 V) + 팔 두 개 명확히

[ScorePanel.tsx](frontend/src/components/ScorePanel.tsx):
- 경로 옵션의 모드 아이콘 크기 **12 → 14px**, 라벨 폰트 10 → 11px

### 가시 변경
- 버스(가로 직사각형 + 채워진 큰 바퀴) vs 지하철(세로 알약 + 트랙) 시각적으로 명확
- 도보 아이콘이 "걷는 사람" 픽토그램으로 더 또렷
